Admin Console Guide

Last Updated: January 15, 2025
Applies to: Account owners and users with the Admin role

1. Accessing the Admin Console

The admin console is available at app.acmecorp.com/admin. Only users with the Owner or Admin role can access the console. Admins see a gear icon in the left sidebar navigation that links directly to it.

2. User Management

2.1 Viewing Users

The Users tab displays all users on the account with their name, email, role, status (active, invited, suspended), last login date, and MFA enrollment status.

2.2 Inviting Users

Click Invite User, enter the email address, select a role (Admin, Member, or Viewer), and optionally assign the user to one or more workspaces. The invitee receives an email with a link that expires after 7 days.

2.3 Changing Roles

Click a user's name, then Edit Role. Role changes take effect immediately. Downgrading an Admin to Member revokes access to the admin console. Only the account Owner can promote a user to Admin.

2.4 Suspending and Removing Users

Suspend: Blocks the user from logging in while preserving their data and workspace memberships. Useful when an employee is on leave or under investigation. Suspended users do not count toward the seat limit for billing.

Remove: Permanently removes the user from the account. Their records are reassigned to the account Owner unless a different assignee is specified. Removal cannot be undone; the user must be re-invited to regain access.

3.2 Setup Steps

  1. Navigate to Admin > Security > Single Sign-On.
  2. Select SAML 2.0 or OIDC.
  3. Enter the metadata URL or upload the metadata XML from your identity provider.
  4. Map identity provider attributes to Acme Corp fields (email, first name, last name, department).
  5. Enable SSO. Optionally, enforce SSO (disabling password-based login for all non-Owner users).
  6. Test the connection using the Test SSO button before enforcing.

3.3 SCIM Provisioning

Enterprise customers can enable SCIM 2.0 provisioning to automatically create, update, and deactivate Acme Corp user accounts from their identity provider. Configure SCIM under Admin > Security > Provisioning. A SCIM bearer token is generated during setup — store it securely in your identity provider's configuration.

4. Audit Log

4.1 Viewing the Audit Log

The audit log is available under Admin > Audit Log. It records all significant actions performed by users and the system:

Event CategoryExamples
AuthenticationLogin, logout, failed login, MFA challenge, password reset
User managementUser invited, role changed, user suspended, user removed
Record operationsRecord created, updated, deleted, restored, exported
ConfigurationSSO configured, API key generated, webhook registered, integration connected
BillingPlan changed, seats added, payment processed, refund issued

4.2 Filtering and Exporting

Use the filters at the top of the log to narrow results by date range, user, event category, or specific action. The audit log is searchable by keyword. Export the filtered results as CSV for compliance reporting.

4.3 Retention

Audit log entries are retained for 90 days on Starter, Pro, and Premium plans. Enterprise customers have extended retention of 1 year in searchable storage plus 6 years in archive, accessible on request. See the Audit Log Retention policy for details.

5. API Key Management

5.1 Generating Keys

Navigate to Admin > API > Keys and click Generate New Key. Assign a name, select a permission scope (read, read-write, or admin), and optionally restrict the key to specific IP addresses. The key is displayed only once — copy it immediately. See the API Authentication documentation for usage details.

5.2 Managing Keys

The API Keys list shows all active keys with their name, scope, creation date, last used date, and creator. You can revoke a key at any time by clicking Revoke. Revoked keys stop working immediately.

5.3 Usage Monitoring

Click a key's name to see usage statistics: requests per day, error rate, and top endpoints by volume. Set up alerts for unusual patterns (spike in 401 errors, requests from new IP addresses) under Admin > API > Alerts.

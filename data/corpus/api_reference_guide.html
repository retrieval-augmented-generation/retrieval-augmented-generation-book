Acme Corp REST API Reference

API Version: v2
Base URL: https://api.acmecorp.com/v2
Last Updated: January 15, 2025

This reference documents all available endpoints in the Acme Corp REST API. All requests must be authenticated using an API key or OAuth 2.0 token. See the API Authentication documentation for details.

1. Overview

1.1 Base URL and Versioning

All API endpoints are served from https://api.acmecorp.com/v2. The version is included in the URL path. When a new major version is released, the previous version remains available for at least 12 months after the deprecation announcement.

The current version (v2) was released in March 2024. The deprecated v1 API remains accessible until March 2025.

1.2 Request Format

All request bodies must be JSON-encoded with the Content-Type: application/json header. Query parameters are used for filtering, sorting, and pagination. Path parameters identify specific resources.

1.3 Response Format

All responses are JSON-encoded. Successful responses return a data field containing the result. Error responses return an error field with a machine-readable code and a human-readable description.

{
  "data": { ... },
  "meta": {
    "request_id": "req_8f2b4a...",
    "timestamp": "2025-01-15T10:30:00Z"
  }
}

Every response includes a unique request_id in the meta object and in the X-Request-ID response header. Include this ID when contacting support.

2. Authentication

See API Authentication for full details. In summary:

  • API key: Pass in the Authorization: Bearer acme_sk_live_... header.
  • OAuth 2.0: Pass the access token in the Authorization: Bearer acme_at_... header.

Unauthenticated requests receive a 401 Unauthorized response.

3. Pagination

3.1 Cursor-Based Pagination

List endpoints return paginated results using cursor-based pagination. Each response includes a pagination object:

{
  "data": [ ... ],
  "pagination": {
    "has_more": true,
    "next_cursor": "eyJpZCI6MTAwfQ==",
    "previous_cursor": null
  }
}

To fetch the next page, pass the next_cursor value as the cursor query parameter:

GET /v2/records?cursor=eyJpZCI6MTAwfQ==&limit=50

3.2 Page Size

Use the limit query parameter to control the number of results per page:

ParameterDefaultMinimumMaximum
limit251100

Requesting a limit above 100 returns a 400 Bad Request error.

3.3 Total Count

List endpoints include a total_count field in the response when the include_count=true query parameter is passed. Counting adds latency for large collections; omit it when not needed.

4.2 Rate Limit Headers

Every response includes rate limit information in headers:

HeaderDescription
X-RateLimit-LimitMaximum requests per minute for the current key/token
X-RateLimit-RemainingRequests remaining in the current window
X-RateLimit-ResetUnix timestamp when the window resets

4.3 Handling 429 Responses

When the rate limit is exceeded, the API returns 429 Too Many Requests with a Retry-After header (in seconds). Implement exponential backoff: wait the Retry-After duration, then double the wait on each consecutive 429 until the request succeeds.

5. Records API

5.1 List Records

GET /v2/records

Returns a paginated list of records in the authenticated account.

Query Parameters

ParameterTypeRequiredDescription
limitintegerNoResults per page (default 25, max 100)
cursorstringNoPagination cursor from a previous response
sortstringNoSort field: created_at, updated_at, name (default: created_at)
orderstringNoasc or desc (default: desc)
statusstringNoFilter by status: active, archived, deleted
created_afterISO 8601NoFilter records created after this timestamp
created_beforeISO 8601NoFilter records created before this timestamp
qstringNoFull-text search across record name and content

Example Response

{
  "data": [
    {
      "id": "rec_a1b2c3",
      "name": "Q3 2025 Revenue Summary",
      "status": "active",
      "created_at": "2025-01-10T14:30:00Z",
      "updated_at": "2025-01-12T09:15:00Z",
      "created_by": "user_x7y8z9",
      "tags": ["finance", "quarterly"]
    }
  ],
  "pagination": {
    "has_more": true,
    "next_cursor": "eyJpZCI6MTAxfQ==",
    "previous_cursor": null
  },
  "meta": {
    "request_id": "req_f3d2a1...",
    "timestamp": "2025-01-15T10:30:00Z"
  }
}

5.2 Get a Record

GET /v2/records/{record_id}

Returns a single record by ID, including its full content.

Path Parameters

ParameterTypeDescription
record_idstringThe record's unique identifier (e.g., rec_a1b2c3)

Response

Returns the full record object including content (the record body), metadata, and attachments array.

Returns 404 Not Found if the record does not exist or the authenticated user does not have access.

5.3 Create a Record

POST /v2/records

Creates a new record. Requires write scope.

Request Body

FieldTypeRequiredDescription
namestringYesRecord name (max 255 characters)
contentstringYesRecord body (max 1 MB)
tagsarray of stringsNoTags for categorization (max 20 tags, max 50 chars each)
metadataobjectNoArbitrary key-value pairs (max 50 keys, max 500 chars per value)

Example Request

POST /v2/records HTTP/1.1
Host: api.acmecorp.com
Authorization: Bearer acme_sk_live_7f3a...
Content-Type: application/json

{
  "name": "Meeting Notes — Product Review",
  "content": "Discussed roadmap priorities for Q2...",
  "tags": ["meetings", "product"],
  "metadata": {
    "department": "Product",
    "meeting_date": "2025-01-14"
  }
}

Returns 201 Created with the full record object. Returns 422 Unprocessable Entity if required fields are missing or validation fails.

5.4 Update a Record

PATCH /v2/records/{record_id}

Updates specific fields of an existing record. Only include fields you want to change. Requires write scope.

Request Body

Same fields as Create, all optional. Omitted fields are left unchanged. To clear a field, pass null.

Returns 200 OK with the updated record. Returns 404 Not Found or 422 Unprocessable Entity as appropriate.

5.5 Delete a Record

DELETE /v2/records/{record_id}

Soft-deletes a record. The record's status is set to deleted and it no longer appears in list responses by default. Deleted records can be restored within 30 days via a support request. After 30 days, the record and its content are permanently purged.

Requires delete scope. Returns 204 No Content on success.

6. Users API

6.1 List Users

GET /v2/users

Returns a paginated list of users in the authenticated account. Requires admin scope.

Query Parameters

ParameterTypeRequiredDescription
limitintegerNoResults per page (default 25, max 100)
cursorstringNoPagination cursor
rolestringNoFilter by role: owner, admin, member, viewer
statusstringNoFilter by status: active, invited, suspended

6.2 Get a User

GET /v2/users/{user_id}

Returns a single user profile. Requires admin scope or the requesting user's own user_id.

6.3 Invite a User

POST /v2/users/invite

Sends an invitation email to a new user. Requires admin scope.

Request Body

FieldTypeRequiredDescription
emailstringYesEmail address of the invitee
rolestringYesadmin, member, or viewer
teamsarray of stringsNoTeam IDs to add the user to upon acceptance

Returns 201 Created. The invitation expires after 7 days. Returns 409 Conflict if the email is already associated with an active user on the account.

6.4 Update a User

PATCH /v2/users/{user_id}

Updates a user's role or status. Requires admin scope. Cannot modify the account owner.

6.5 Remove a User

DELETE /v2/users/{user_id}

Removes a user from the account. Their records are reassigned to the account owner unless a reassign_to parameter is specified. Requires admin scope. Returns 204 No Content.

7. Search API

7.1 Search Records

POST /v2/search

Full-text and semantic search across records in the authenticated account.

Request Body

FieldTypeRequiredDescription
querystringYesSearch query (max 500 characters)
modestringNokeyword, semantic, or hybrid (default: hybrid)
limitintegerNoMaximum results (default 10, max 50)
filtersobjectNoFilter by tags, created_after, created_before, status, metadata

Example Request

POST /v2/search HTTP/1.1
Host: api.acmecorp.com
Authorization: Bearer acme_at_8f2b...
Content-Type: application/json

{
  "query": "revenue forecast Q3",
  "mode": "hybrid",
  "limit": 5,
  "filters": {
    "tags": ["finance"],
    "created_after": "2025-01-01T00:00:00Z"
  }
}

Example Response

{
  "data": [
    {
      "id": "rec_a1b2c3",
      "name": "Q3 2025 Revenue Summary",
      "snippet": "...projected revenue of $38.2M for Q3, representing 16% YoY growth...",
      "score": 0.94,
      "highlights": ["revenue", "Q3"]
    }
  ],
  "meta": {
    "request_id": "req_b4c5d6...",
    "search_time_ms": 47,
    "total_results": 12,
    "timestamp": "2025-01-15T10:35:00Z"
  }
}

Results are ranked by relevance score (0.0 to 1.0). In hybrid mode, scores combine keyword (BM25) and semantic (vector similarity) signals.

8. Webhooks API

8.1 Register a Webhook

POST /v2/webhooks

Request Body

FieldTypeRequiredDescription
urlstringYesHTTPS endpoint to receive events (must be publicly accessible)
eventsarray of stringsYesEvents to subscribe to: record.created, record.updated, record.deleted, user.invited, user.removed
secretstringNoShared secret for signature verification. Auto-generated if not provided.

Returns 201 Created with the webhook object including the id and secret.

8.2 Webhook Delivery

Events are delivered as POST requests to the registered URL with a JSON body:

{
  "id": "evt_m3n4o5",
  "type": "record.updated",
  "created_at": "2025-01-15T10:40:00Z",
  "data": {
    "record_id": "rec_a1b2c3",
    "changes": ["content", "tags"]
  }
}

Each delivery includes an X-Acme-Signature header containing an HMAC-SHA256 signature of the request body using the webhook secret. Verify this signature to ensure the request is from Acme Corp.

Retry policy: If your endpoint does not respond with a 2xx status within 10 seconds, the delivery is retried up to 5 times with exponential backoff (10s, 30s, 90s, 270s, 810s). After 5 failures, the webhook is marked as failing and an email notification is sent to the account owner.

8.3 List Webhooks

GET /v2/webhooks

Returns all registered webhooks for the authenticated account. Requires admin scope.

8.4 Delete a Webhook

DELETE /v2/webhooks/{webhook_id}

Removes a webhook subscription. Pending deliveries are cancelled. Returns 204 No Content.
